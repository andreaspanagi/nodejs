# 📁 Complete File Guide

## What This Document Does

This guide explains EVERY file in the project and what it does! Use this as a reference when you're wondering "What is this file for?"

---

## 📂 Root Level Files

### `app.js` ⭐ MAIN FILE
**What it does:**
- The boss of the application!
- Starts the web server
- Connects all routes
- Sets up middleware
- Handles 404 errors

**Think of it as:** The restaurant manager who opens the doors and coordinates everything

---

### `package.json`
**What it does:**
- Lists all the packages (libraries) the project needs
- Defines scripts like `npm start`
- Contains project metadata (name, version, author)

**Think of it as:** A shopping list of ingredients needed to run the app

---

### `package-lock.json`
**What it does:**
- Locks exact versions of all packages
- Ensures everyone gets the same versions
- Auto-generated by npm

**Think of it as:** A detailed receipt showing exact versions of everything

---

### `.gitignore`
**What it does:**
- Tells Git which files NOT to track
- Usually ignores `node_modules/` and sensitive files

**Think of it as:** A "Do Not Touch" list for version control

---

## 📂 `/controllers` - The Waiters

Controllers handle requests and send responses. They're the middlemen between routes and models!

### `controllers/shop.js`
**What it does:**
- Handles customer-facing actions
- Functions for: home, products, cart, orders, checkout
- Gets data from models, sends to views

**Functions:**
- `getIndex` - Show home page
- `getProducts` - Show all products
- `getProduct` - Show one product detail
- `getCart` - Show shopping cart
- `postCart` - Add item to cart
- `postCartDeleteProduct` - Remove item from cart
- `getOrders` - Show orders (placeholder)
- `getCheckout` - Show checkout (placeholder)

**Think of it as:** Waiter handling customer requests

---

### `controllers/admin.js`
**What it does:**
- Handles admin actions
- Functions for: add, edit, delete products
- Manages product catalog

**Functions:**
- `getAddProduct` - Show add product form
- `postAddProduct` - Save new product
- `getEditProduct` - Show edit product form
- `postEditProduct` - Update product
- `getProducts` - Show admin product list
- `postDeleteProduct` - Delete a product

**Think of it as:** Manager handling inventory

---

### `controllers/error.js`
**What it does:**
- Handles error pages
- Currently just 404 (page not found)

**Functions:**
- `get404` - Show 404 error page

**Think of it as:** Security guard who says "You can't go there"

---

## 📂 `/models` - The Recipe Books

Models handle data storage and retrieval. They work with files to save/load data!

### `models/product.js`
**What it does:**
- Manages product data
- Saves to/reads from `data/products.json`
- CRUD operations: Create, Read, Update, Delete

**Methods:**
- `constructor` - Create product object
- `save()` - Add new or update existing product
- `static fetchAll(cb)` - Get all products
- `static findById(id, cb)` - Find one product
- `static deleteById(id)` - Delete a product

**Think of it as:** Product catalog manager

---

### `models/cart.js`
**What it does:**
- Manages shopping cart data
- Saves to/reads from `data/cart.json`
- Tracks items and total price

**Methods:**
- `static addProduct(id, price)` - Add item to cart
- `static deleteProduct(id, price)` - Remove item from cart
- `static getCart(cb)` - Get cart contents

**Think of it as:** Shopping basket keeper

---

## 📂 `/routes` - The Menu

Routes define URL paths and which controller handles each path!

### `routes/shop.js`
**What it does:**
- Defines customer-facing routes
- Maps URLs to shop controller functions

**Routes:**
- `GET /` → Home page
- `GET /products` → Product list
- `GET /products/:productId` → Product detail
- `GET /cart` → Shopping cart
- `POST /cart` → Add to cart
- `POST /cart-delete-item` → Remove from cart
- `GET /orders` → Orders page
- `GET /checkout` → Checkout page

**Think of it as:** Customer menu

---

### `routes/admin.js`
**What it does:**
- Defines admin routes (all start with `/admin`)
- Maps URLs to admin controller functions

**Routes:**
- `GET /admin/add-product` → Add product form
- `POST /admin/add-product` → Save new product
- `GET /admin/products` → Admin product list
- `GET /admin/edit-product/:productId` → Edit product form
- `POST /admin/edit-product` → Update product
- `POST /admin/delete-product` → Delete product

**Think of it as:** Manager's menu

---

## 📂 `/data` - The Storage Room

JSON files that store application data!

### `data/products.json`
**What it does:**
- Stores all product information
- Array of product objects
- Each has: id, title, imageUrl, description, price

**Example:**
```json
[
  {
    "id": "0.12345",
    "title": "Book",
    "imageUrl": "http://...",
    "description": "A nice book",
    "price": 19.99
  }
]
```

---

### `data/cart.json`
**What it does:**
- Stores shopping cart contents
- Object with products array and total price
- Each product has: id, qty (quantity)

**Example:**
```json
{
  "products": [
    {"id": "0.12345", "qty": 2}
  ],
  "totalPrice": 39.98
}
```

## 🎓 Understanding File Types

### `.js` files
- JavaScript code
- Run on server (Node.js)
- Controllers, models, routes, utilities

### `.ejs` files
- Embedded JavaScript templates
- Create HTML with dynamic data
- Views (what users see)

### `.json` files
- JavaScript Object Notation
- Store data as text
- Configuration and data storage

### `.css` files
- Cascading Style Sheets
- Make pages look pretty
- Colors, layout, fonts

## 🚀 How Files Work Together

### Example: Viewing Product List

```
1. User visits /products
   ↓
2. app.js receives request
   ↓
3. routes/shop.js matches route
   ↓
4. controllers/shop.js → getProducts function
   ↓
5. models/product.js → fetchAll (reads data/products.json)
   ↓
6. Product data returned to controller
   ↓
7. views/shop/product-list.ejs creates HTML
   ↓
8. Uses: views/includes/head.ejs (header)
         views/includes/navigation.ejs (menu)
         views/includes/end.ejs (footer)
         public/css/main.css (styling)
         public/css/product.css (product styling)
   ↓
9. HTML sent to browser
   ↓
10. User sees beautiful product list! 🎉
```

## 💡 Pro Tips

### Finding What You Need

**Looking for a page?**
- Check `views/` folder

**Looking for logic?**
- Check `controllers/` folder

**Looking for data handling?**
- Check `models/` folder

**Looking for routes?**
- Check `routes/` folder

**Looking for styles?**
- Check `public/css/` folder

**Looking for stored data?**
- Check `data/` folder

### File Naming Conventions

- **kebab-case**: `product-list.ejs`, `add-to-cart.ejs`
- **camelCase**: `getProducts`, `postCart`
- **PascalCase**: `Product`, `Cart` (for classes)

### Comments in Code

Look for these in files:
```javascript
// Single line comment
/* Multi-line
   comment */
```

They explain what code does!
